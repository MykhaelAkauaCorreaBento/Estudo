(---------------------Passo 1: Legibilidade do Código---------------------)
Aprendizado: O Python expressa ideias com clareza
Python foi projetado para que o código se pareça com instruções naturais. Cada linha expressa uma ação clara, sem 
símbolos ou estruturas desnecessárias.

Veja o exemplo abaixo, que calcula a média de um aluno e exibe se ele foi aprovado:

nota1 = 7
nota2 = 8
media = (nota1 + nota2) / 2

if media >= 6:
    print("Aprovado")
else:
    print("Reprovado")
Perceba que o código acompanha o raciocínio de forma natural: duas notas são definidas, a média é calculada, uma 
condição é verificada e um resultado aparece. Cada parte está ali porque representa uma etapa da solução, nada a mais.

É isso que torna o Python legível. Quem escreve consegue expressar a ideia com clareza, e quem lê consegue acompanhar 
o que está acontecendo sem precisar decifrar a linguagem.

(---------------------Aprendizado 2: Python comparado a outras linguagens---------------------)
O mesmo programa escrito em C teria a seguinte forma:

#include <stdio.h>

int main() {
    int nota1 = 7;
    int nota2 = 8;
    float media = (nota1 + nota2) / 2.0;

    if (media >= 6) {
        printf("Aprovado\n");
    } else {
        printf("Reprovado\n");
    }
    return 0;
}
A lógica é a mesma, mas o C exige elementos que não descrevem a solução: declaração explícita de tipos, importação de
biblioteca, função main, chaves para abrir e fechar blocos e printf para exibir resultados. Esses elementos existe
porque a linguagem exige, não porque a solução precisa deles.

Em Python, nada disso é necessário. O código começa na primeira linha útil, cada instrução faz exatamente o que parece
fazer, e o resultado aparece sem cerimônia. Essa diferença não é apenas estética, ela significa menos tempo decifrando
a linguagem e mais tempo resolvendo o problema.

(---------------------Passo 2: Indentação obrigatória---------------------)
Aprendizado: A indentação define a estrutura do código
Indentação é o espaço no início de uma linha de código. Em Python, esses espaços não são apenas uma convenção de 
organização, eles definem quais linhas pertencem a cada bloco.

Veja o exemplo abaixo:

if numero == 3:
    print("Encontrei")
A linha print("Encontrei") está indentada porque pertence ao if. Ou seja, ela só será executada quando a condição 
for verdadeira.

Sem a indentação correta, o Python retorna um erro:

if numero == 3:
print("Encontrei")

Retorno:

IndentationError: expected an indented block after 'if' statement
Esse comportamento é intencional. Ao tornar a indentação obrigatória, Python garante que todo código seja escrito de 
forma organizada desde o início.

(---------------------Aprendizado 2: Indentação em blocos encadeados---------------------)
A indentação também se aplica quando há blocos dentro de blocos. Cada nível de profundidade exige um recuo adicional.

Veja o exemplo abaixo, que percorre uma lista e identifica um valor específico:

for numero in [1, 2, 3]:
    print(numero)
    if numero == 2:
        print("Encontrei o 2!")
O print(numero) e o if pertencem ao for, por isso estão no primeiro nível de indentação. Já o print("Encontrei o 2!") 
pertence ao if, que por sua vez está dentro do for, por isso recebe um recuo a mais.

Cada nível de indentação reflete um nível de profundidade na lógica do programa.

(---------------------Aprendizado 3: Python comparado a outras linguagens---------------------)
Em linguagens como JavaScript, a indentação não é obrigatória. Os blocos de código são delimitados por chaves {}, e o 
programa funciona independente da organização visual.

Veja o mesmo exemplo escrito em JavaScript, sem indentação:

for (let numero of [1, 2, 3]) {
print(numero)
if (numero === 2) {
print("Encontrei o 2!")
}
}
O código acima funciona normalmente em JavaScript. Em Python, o equivalente causaria um IndentationError imediatamente.

Essa diferença tem uma consequência prática: em Python, um código mal indentado simplesmente não roda. Em JavaScript,
ele roda, mas pode ser difícil de ler e dar manutenção. A obrigatoriedade do Python elimina esse problema desde o início.

(---------------------Passo 3: Tipagem dinâmica---------------------)
Aprendizado: Variáveis sem declaração de tipo
Em Python, para guardar um valor em uma variável, basta atribuí-lo diretamente. A linguagem identifica o tipo 
automaticamente, sem que você precise informar se é um número, um texto ou outro tipo de dado.

Veja o exemplo:

numero = 10
texto = "Aprovado"
decimal = 7.5
verdadeiro = True

print(numero)
print(texto)
print(decimal)
print(verdadeiro)
Nenhuma linha declara o tipo da variável. Python observou o valor atribuído e determinou o tipo sozinho: 10 é um
número inteiro, "Aprovado" é um texto, 7.5 é um número decimal e True é um booleano, um valor que representa 
verdadeiro ou falso.

Esse comportamento se chama tipagem dinâmica. O código fica mais direto e menos burocrático, porque você começa a 
trabalhar com os dados sem precisar configurar as variáveis antes.

(---------------------Aprendizado 2: Python comparado a outras linguagens---------------------)
Em linguagens de tipagem estática, como Java, cada variável precisa ter seu tipo declarado antes de receber um valor.
Veja o mesmo exemplo escrito em Java:

int numero = 10;
String texto = "Aprovado";
double decimal = 7.5;
boolean verdadeiro = true;

System.out.println(numero);
System.out.println(texto);
System.out.println(decimal);
System.out.println(verdadeiro);
O resultado é o mesmo, mas o Java exige que cada variável seja declarada com seu tipo antes de receber um valor: int 
para inteiro, String para texto, double para decimal e boolean para verdadeiro ou falso. Se o tipo estiver errado, 
o programa nem compila.

Em Python, nada disso é necessário. Você atribui o valor e segue em frente. Isso não significa que os tipos não existem,
 significa que o Python cuida disso por você, permitindo que o foco fique na solução.

(---------------------Aprendizado 3: Tipagem opcional em Python---------------------)
Embora o Python identifique os tipos automaticamente, ele também permite que o tipo de cada variável seja declarado
explicitamente no código. Esse recurso é chamado de type hints, anotações de tipo.

Veja o mesmo exemplo anterior com type hints:

numero: int = 10
texto: str = "Aprovado"
decimal: float = 7.5
verdadeiro: bool = True

print(numero)
print(texto)
print(decimal)
print(verdadeiro)
As anotações como int, str, float e bool indicam qual tipo é esperado para cada variável. O comportamento do
código continua idêntico, o Python não impede que o valor seja de outro tipo. A declaração serve como uma indicação
para quem lê o código e para ferramentas de análise que podem alertar sobre inconsistências antes da execução.

O uso de type hints é opcional. Você pode escrever código Python sem eles e o resultado será o mesmo.
