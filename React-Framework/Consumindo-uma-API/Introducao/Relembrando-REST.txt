(--------------------------------Relembrando REST--------------------------------)

Como já estudamos anteriormente, REST é um conjunto de regras que define como a API deve ser
implementada.

Essas regras servem para que as APIs sigam um padrão simples de usar, além de trazer
diversos benefícios tais como: performance e escalabilidade para a API.

Na aula anterior lembramos que um sistema Fullstack é composto de um Front-end Back-end.
Ambos se conectam através de APIs.

exemplo_1

A API é um padrão de integração. Qualquer Front-end pode se conectar a qualquer Back-end, desde
que adotem o mesmo padrão de API.

exemplo_2

Existem vários tipos de API. Nesta trilha, toda vez que falamos "API", estamos nos referindo a um
tipo específico de API, que se chama API RESTful, ou simplesmente API REST.

Os termos API RESt ou API RSTful são sinônimos.
A API do tipo REST é o padrão adotado atualmente pelo mercado.

Desde forma, toda vez que você ler "API", fica subentendido que estamos nos referindo a APIs
do tipo REST, o padrão de mercado.
