Simulando problemas

Agora, veremos como tratar outros tipos de problemas tais como erro 404, erro 500 e timeout.

Porém, é necessário antes conhecer um tipo de ferramenta conhecida como mockup, que vai nos
ajudar a testar esse tipo de erro.

Os mockups são como chamamos dados fictícios que usamos em uma aplicação.

Nesse caso, a ferramenta de mockup vai nos ajudar a criar endpoints fictícios que simulam erros
dentro de uma API.



Vamos avançar no tratamento de erros vendo como lidar com o erro 404 Not Found.

No contexto de uma API o erro 404 Not Found ocorre quando um endpoint ou recurso não existe
(url não encontrada/ UF não encontrada).
