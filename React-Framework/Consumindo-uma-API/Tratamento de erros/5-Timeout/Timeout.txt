Timeout
Ótimo! Você melhorou sua aplicação tratando os erros com códigos de status.

Nós vimos previamente que a API funciona de forma assíncrona, ou seja, não
sabemos quando tempo vai levar parar retornar uma resposta.

O timeout nada mais é tempo que o Front-end vai esperar pela resposta da API.

VAmos dizer que estamos com a internet lenta, e ao abrir o site vemos uma
tela em branco por mais de 1 minuto até os dados carregarem.

Isso é uma experiência ruim, 1 minuto é tempo demais e o usuário não sabe
se está carregando ou se deu erro.

Pra testar isso, remova o parâmetro mock_status e inclua o parâmetro mock_delay=30000
