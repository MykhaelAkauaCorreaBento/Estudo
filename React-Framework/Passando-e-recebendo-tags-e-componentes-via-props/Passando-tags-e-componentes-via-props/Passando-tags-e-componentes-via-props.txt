Passando tags e componentes via props

Na aula anterior vimos que as tags e componentes podem ser passadas via props para outros
componentes de uma aplicação React. Os elementos enviados são representadas por meio do termo
props.children.
O props.children é, basicamente, uma propriedade que possibilita a passagem de elementos, sejam
eles componentes ou tags, como filhos para um componente pai. 

Passando tags e componentes via props

Até o momento, enviamos dados para um componente, passando props diretamente na tag.

Componente ItemLista

<ItemLista
    signo={'Capricornio'}
    dataInicio={'21/01'}
    dataFim={'19/02'}
    imagem={Capricornio}
/>

exemplo_1.png

Contudo, para enviar um conteúdo (tags HTML ou componentes) vamos precisar de duas tags, sendo
uma de abertura e a outra de fechamento.

exemplo_2.png

Todo conteúdo (tags ou componentes) que será passado via props será inserido entre as tags de
abertura e fechamento.

<NomeDoComponente>

  <h1>Este é um título</h1>
  <OutroComponente />

</NomeDoComponente>

exemplo_3.png

Com isso, o conteúdo ficará acessível no componente através do termo props.children.

export default function NomeDoComponente(props) {
  return(
    <div>{props.children}</div>
  );
}
