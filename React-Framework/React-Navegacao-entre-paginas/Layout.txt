Layout

layout.js é um arquivo que vem, por padrão, em uma aplicação React quando essa usa o App Router.
Imagine um site no qual alguns componentes (Topo ou Rodape) se repetirão em todas as páginas
Com o layout.js você pode inserir esses componentes apenas uma vez, e todas as páginas vão
 exibir automaticamente esses componentes.
 Imagine um site em que as suas páginas exibem os mesmos componentes (Topo ou Rodape).
 Nesse caso, os componentes serão inseridos diretamente nas páginas.

 Com o uso do layout.js, você pode exibir esses componentes sem precisar
 inseri-los diretamente nas páginas.
 exemplo:
 foto29.jpeg

 Por fim, embora não seja obrigatório, o uso do layout.js auxilia na redução da escrita de códigos,
 otimizando o desenvolvimento do projeto.

 Observe que o uso do layout.js não muda o comportamento padrão SPA.


 Agora vamos analisar o arquivo layout.js e o código padrão que vem nele.
exemplo:
 foto30.jpeg

 Observe que o layout.js recebe o estilo global da aplicação.
 Veja que o layout.js é basicamente um componente de nome RootLayout. Esse componente é a
 estrutura básica do site.
 O RootLayout recebe todo o conteúdo do site parâmetro chamado "children", aplicado na tag <body>.
 exemplo:
 foto30.jpeg

import Inter from "next/font/google";
import "./globals.css";

const inter = Inter({ subsets: ["Latin"] });

export const metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body className={inter.className}></body>
    </html>
  );
}





Usando meta tags no layout.js


Observe que no arquivo layout.js existe uma constante de nome metadata.
Essa constante é responsável por receber informações como o título e a descrição do site.

exemplo:
 foto31.jpeg


As informações contidas em metadata tem a mesma finalidade que as meta tags
 utilizadas no componente <Head>.

exemplo:
foto32.jpeg

Veja que, ao criarmos uma aplicação que usa o App Router, o arquivo principal não vem
com as meta tags incluídas.

exemplo:
foto33.jpeg


Nesse caso, como as meta tags não estão incluídas, a constante metadata é utilizada como
alternativa para definir o título e a descrição do site.

exemplo:
foto34.jpeg

Lembrando que as meta tags mais utilizadas em uma página web são:

title, description, charset, viewport e robots.

exemplo:
foto35.jpeg

Em aplicações que usam o App Router, o React define por padrão o valor das meta tags
charset e viewport.
Ou seja, os valores de charset e viewport não precisam ser alterados.

O valor padrão adotado pelo App Router para o charset e o viewport é comum para quase todos os
sites.

exemplo:
foto36.jpeg

Alterar os valores padrão do charset e viewport é um processo mais específico.
Para não sair do foco principal do curso, não vamos nos aprofundar nesse detalhamento.

Além de tit/e e description, podemos também adicionar outras metas tags, como o robots:
Veja abaixo a sintaxe usada para adicionar robots:

exemplo:
foto37.jpeg

export const metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
  robots: "index, follow",
};
